Lerneinheit 1: HTTP, URI und HTML
1.1. Fachliche Argumentation über Erfolgsprinzipien des WWW (2 Punkte)
Mit welchen fachlichen Argumenten wurde das WWW-Proposal von TBL abgelehnt?
Ein Argument von der Hypertextcommunity war, dass das Konzept von TBL ein Rückschritt gewesen wäre, da sie schon weiter in der Wissenschaft vorangeschritten seien und mehr erreicht haben. Zudem bemängeln sie das mögliche Auftreten der "broken links", also Links, die ins Leere zeigen und auf nichts mehr verweisen.
Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?
Argumente für den Erfolg sind die Eigenschaften der Offenheit, Dezentralität und Evolvierbarkeit.
Was wäre der Preis für die garantierte Verhinderung von “broken links”?
Ein Nachteil der referentiellen Integrität ist der erhöhte Bedarf an Rechenressourcen.
1.2. HTTP (2 Punkte)
Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?
HTTP Status Code 200 bedeutet, dass die Anfrage erfolgreich war.
Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?
HTTP Status Code 301 bedeutet, dass die angefragte Seite dauerhaft auf eine andere Seite verschoben wurde und der Link ausgetauscht werden sollte.
Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?
HTTP Status Code 400 bedeutet, dass der Server die Anfrage nicht bearbeiten kann oder bearbeiten wird. Man kann die angefragte URL auf Fehler überprüfen, den Cache leeren, das System neu starten, die Internetverbindung überprüfen oder Browser Erweiterungen deaktivieren.
Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?
HTTP Status Code 403 bedeutet, dass der Zugriff vom Server verwehrt wurde. Man kann prüfen, ob man die benötigten Rechte zum Zugriff besitzt oder ob man sich richtig autorisiert hat. Auch hier kann man den Cache leeren oder das System neu starten.
In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performanzgründen soll die Abfrage jedoch cacheable sein. Wie könnten Sie dafür eine Lösung angehen?
Man könnte die Daten für eine OPTIONS Anfrage über eine GET Anfrage abrufbar machen und diese GET Anfrage dann cachen.
1.3. HTML-Literatur lesen und Fragen beantworten (2 Punkte)
Was ist HTML?
HTML ist eine Sprache, mit der man Bilder, Links und Text strukturiert. Mit HTML wird der Inhalt im WWW dargestellt.
Wie kann man eine geschachtelte geordnete Liste der Schachtelungstiefe 3 erzeugen?
Man kann man eine geschachtelte geordnete Liste der Schachtelungstiefe 3 erzeugen, indem man in die li Listenelemente wieder eine geordnete Liste mit ol einfügt. Dort drin fügt man wieder eine geordnete Liste in dem Listenelement ein und wiederholt den Einfügeprozess. Den Einfügeprozess wiederholt man ingesamt dann 3 Mal.
Wie ist eine HTML-Tabelle aufgebaut?
Eine HTML-Tabelle wird mit dem table Tag eingeleitet. Für jede Zeile in der Tabelle benutzt man ein tr Tag. In diesem Tag wird der Inhalt jeder Zelle in ein td Tag geschrieben. Zusätzlich kann man noch innerhalb des td Tags ein th Tag benutzen für die Kopfzeile.
Welche Konventionen sollte man bei Pfaden und Dateinamen beachten?
Die Namen aller Dateien sollten kleingeschrieben werden, sowie nur Buchstaben, Zahlen, Bindestriche und Unterstriche enthalten. Dateien sollten nach Typ in entsprechende Ordner sortiert werden, wie 'img' für Bilder. Dateien sollten dem Inhalt entsprechend benannt werden, wie 'index' für die Startseite und Einstiegspunkt.
Wie baut man in HTML ein Menü?
Für ein Menü eignet sich eine ungeordnete Liste mit dem u Tag. Die Menüeinträge werden dann von li Tags als Listenelement umschlossen. Wenn damit ein Navigationsmenü implementiert wird, dann umschließt man die Liste bzw. den Inhalt noch mit einem nav Tag.
Welche Attribute sollte man bei Bildern wie verwenden?
Das 'src' Attribut wird für die Quelle des Bildes durch eine URL verwendet. 'height' und 'width' werden für die Angaben der Breite und Höhe verwendet. Beim 'alt' Attribut wird Text angegeben, der angezeigt wird, falls das Bild nicht geladen werden kann.
